# 采用状态机+数据库表实现轻量工作流，不引入 BPMN 引擎

请假审批需要"发起→流转→审批→查询→归档"的完整闭环，但本项目主动放弃"可视化流程配置"这一加分点（requirement.txt 第 20 行）。因此不引入 Flowable/Activiti 等 BPMN 引擎，而是用业务表中的状态字段（待审批→审批中→已通过/已驳回→已归档）+ 服务端状态流转逻辑来驱动流程。

## Considered Options

- **Flowable/Activiti + bpmn-js**：唯一能命中"可视化配置工作流"的方案，但学习曲线陡、需自动建 ACT_ 系列表，且其最大卖点（可视化）已被主动放弃，性价比骤降。
- **搬运 RuoYi-Flowable 等现成集成方案**：开箱即用，但与当前 plate-exam 融合成本高，答辩归属难解释。
- **状态机 + 数据库表（采纳）**：无新依赖、完全可控、易讲清，能完整覆盖闭环硬要求。

## Consequences

- 工作流"流转规则"写在服务端代码中，新增审批环节需改代码而非可视化拖拽。
- 失去"可视化配置"加分点，但保住了被反复强调的工作流闭环核心得分项。
